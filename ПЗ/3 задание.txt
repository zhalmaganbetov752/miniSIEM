Задание 3. Исходные данные к работе. В качестве исходных данных для проектирования и реализации программного комплекса mini SIEM использовались сведения, полученные в ходе прикладных исследований, выполненных на предприятии — в филиале АО «ТрансТелеКом» (г. Астана). На основании анализа эксплуатируемой ИТ-инфраструктуры рабочих станций, действующих политик аудита Microsoft Windows и принятых регламентов работы с событиями безопасности был сформирован следующий перечень исходных данных:

целевая операционная система — Microsoft Windows 10 x64;
язык программирования — Python версии 3.11;
библиотека построения графического интерфейса — PySide6 (Qt 6);
встраиваемая система управления базами данных — SQLite 3 с включёнными режимом WAL (Write-Ahead Logging) и поддержкой внешних ключей (foreign keys);
средство упаковки автономного исполняемого файла — PyInstaller;
средства доступа к подсистеме журналирования Windows — пакет pywin32 совместно с Windows Eventing API (Evt API);
источники событий — журналы Microsoft Windows Event Log: Security, System, Application и Microsoft-Windows-Sysmon/Operational, а также архивные файлы в формате EVTX;
формат описания правил детектирования — YAML.
Достоверность приведённых исходных данных обеспечивается их получением непосредственно в производственных условиях предприятия: состав журналируемых каналов, версии операционной системы и системного программного обеспечения, а также требования к режимам хранения и обработки событий зафиксированы по результатам обследования рабочих мест и согласованы с действующей практикой эксплуатации средств информационной безопасности. Практический характер используемой информации подтверждается её соответствием реальным условиям функционирования рабочих станций оператора связи и применимостью к типовым сценариям мониторинга событий безопасности конечных точек.
