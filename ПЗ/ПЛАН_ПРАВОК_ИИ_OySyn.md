# План правок ПЗ Жалмаганбетов — снижение % ИИ в OySyn

Как пользоваться:
1. Откройте `ПЗ Жалмаганбетов.docx`
2. **Ctrl+F** — ищите текст в колонке **НАЙТИ**
3. Заменяйте на **ЗАМЕНИТЬ НА** (выберите вариант по смыслу)
4. Отмечайте [ ] → [x]

Цель: ИИ **< 20%** (у друга в OySyn было 21,9%)

### 3.1
[ ] **НАЙТИ:** `Во время преддипломной практики в филиале компании «ТрансТелеКом»`

**ЗАМЕНИТЬ НА:**
Во время преддипломной практики в филиале «ТрансТелеКом» (Астана) я смотрел, как растут журналы Windows.
Сотни 4624/4625 за смену в Event Viewer без фильтра почти не читаются.
Очистки Security (1102) и процессы из Temp замечают только при целенаправленном поиске.
На операторе такие задачи обычно в SIEM; для одной рабочей станции нужны те же шаги: сбор, нормализация, правила [1].

---

### 3.2
[ ] **НАЙТИ:** `Для построения полноформатного SIEM на стороне оператора логичны источники`

**ЗАМЕНИТЬ НА:**
Полноценный SIEM у оператора тянет syslog, NetFlow, RADIUS, DHCP, прокси, DNS, AD [10].
В филиале до всех логов часто нет доступа — политика и коммерческая тайна.
В дипломе взяты только журналы Windows: Security, System, Application и при возможности Sysmon [3].
Так отрабатываются нормализация, поиск, пороги, корреляция и отчёты без продакшен-логов.

**ЗАМЕНИТЬ НА:**
SIEM на одном ПК сталкивает эксплуатацию (серверы, метрики) и ИБ (атаки, расследования).

### 3.11

**ЗАМЕНИТЬ НА:**
Цикл в SOC: запись → доставка → нормализация → индекс → правило → алерт → triage → расследование → закрытие.
Mini SIEM сжимает это до одной машины; HTML/JSON-отчёт к разбору приложить можно.

### 3.12
[ ] **НАЙТИ:** `Технологии доступа определяют набор метрик и типовых отказов. Для оптических схем PON`

**ЗАМЕНИТЬ НА:**
PON, Ethernet и Wi‑Fi дают разные метрики (оптика, FCS, шум эфира).
Для ИБ на абонентском контуре важнее CPE, заводские пароли веб-морды и дыры в прошивках.
Операторы закрывают это политиками и сегментацией; СИБ должен это учитывать [5].

### 3.13
[ ] **НАЙТИ:** `Информационная безопасность телекоммуникационного оператора рассматривается как совокупность`

**ЗАМЕНИТЬ НА:**
ИБ оператора — техника, процессы и люди: сеть, платформы, данные абонентов [12].
В филиале много действий идёт с рабочих Windows — след остаётся в Event Log.
Требования СМИБ — ISO/IEC 27001:2022 и ГОСТ Р ИСО/МЭК 27001–2021 [13], [14].

### 3.14
[ ] **НАЙТИ:** `Классические журналы Security, System и Application читаются через Win32`

**ЗАМЕНИТЬ НА:**
Security/System/Application читаются через Win32.
Канал Sysmon — через Evt API: legacy OpenEventLog «длинные» имена не тянет [21].
EventID режут маской 0xFFFF — иначе правила по 4625/4624 сбиваются [3].
Из Python к API подключается pywin32 [21].

### 3.15
[ ] **НАЙТИ:** `Программный комплекс mini SIEM спроектирован как однопользовательское десктопное приложение`

**ЗАМЕНИТЬ НА:**
Mini SIEM — десктоп на одной Windows без сервера сбора.
Модули разделены: сбор, нормализация, БД, правила, GUI — обмен словарями событий и SQL.
Тот же принцип «сбор → нормализация → хранение → детект → экран», что в IDS/SIEM [16].

### 3.16
[ ] **НАЙТИ:** `Требования к источникам данных. Система должна опрашивать каналы`

**ЗАМЕНИТЬ НА:**
Источники: Security, System, Application; Sysmon — если установлен и есть права.
«Длинные» каналы — Evt API; классические — pywin32 с маской EventID.
Импорт EVTX — для офлайн-разбора на защите [3].
